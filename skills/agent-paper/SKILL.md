# Agent Paper

分析学术论文并生成结构化总结报告。

## 工作流程

当用户提供论文时（通过 /agent-paper 命令或直接提供）：

1. **读取论文**
   - 如果输入是PDF路径：读取PDF内容
   - 如果输入是URL：获取并提取论文内容
   - 如果输入是文本：直接使用提供的内容

2. **提取核心信息**
   - 论文元数据（标题、作者、发表时间、发表平台、链接）
   - 研究问题和研究动机
   - 提出的方法/方案
   - 关键创新点/贡献
   - 实验结果和发现

3. **生成报告**，使用下方的模板

4. **保存并打开**
   - 文件名格式：`{年份}-{第一作者}-{短标题}.md`
   - 保存路径：当前执行目录（即运行命令的目录）
   - 打开文件供用户查看

## 报告模板

```markdown
# {论文标题}

## 论文信息

- **标题**：{完整标题}
- **作者**：{作者列表}
- **发表时间**：{年份/日期}
- **发表平台**：{会议/期刊/平台}
- **链接**：{如果是URL输入，附上原始链接}

## 研究问题

{论文要解决什么问题？背景和动机是什么？}

## 解决方法

{论文提出了什么方案？关键技术细节是什么？}

## 创新点

1. {主要贡献1}
2. {主要贡献2}
3. {主要贡献3}

## 实验结果

### 实验设置
{数据集、评估指标、基线方法}

### 主要结果
{关键性能数据和对比结果}

### 消融实验
{消融实验揭示了什么结论？}

## 总结

{整体评价和核心收获}
```

## 输出要求

- 准确反映论文内容
- 使用清晰简洁的语言
- 包含具体细节（数字、数据集名称、指标）
- 如果信息缺失，标注为"未明确说明"
